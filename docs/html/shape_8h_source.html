31// Shape object, responsible for generating and storing 2D Shapes.
32// The data here is used for both Rendering and Physics.
33// This Shape structure is made to be malliable, and allow for a lot of freedom.
34// Do note that having a lot of verticies is performance intensive, and its not suggested to run isConvex too much
35// if possible know if the shape is convex or not, or just simplify the collisions to always be or not be convex
36//
37// A dynamic array of Verticies. Normalized between [0, 1], (0, 0) being bottom left.
38// This way the size of your Object doesn't matter, and can be transposed to whatever Size freely.
39// Each vertici needs to be processed anyways to place onto the screen, and for collision checks. So it really shouldn't add any overhead.
40// ^ Requires one loop through the Array to be true, so gotta handle rendering and collisions in that one for loop.
41// ^^ Simply find the proper pixel positions in Fixed Update, but only render them in the variable update.
42// Layout: [Vector, Pose, uint32, bool]
43// Size: 85(88), 53(56)
44typedef struct Shape {
45 struct VertexVector normalized_vertices; // Dynamic Array of Normalized Verticies.
46 struct PoseVector vertex_poses; // Dynamic Array of Vertex Positions.
47 struct Pose size; // Size of Shape, used to Transpose the Normalized Verticies.
48 uint32_t color; // *Until Textures become a thing.
49 bool convex; // Whether a Shape is Convex or Concave.
50} Shape;
